## Common Components

**Check `${IP_COMMON_PATH}/rtl/` before writing any new RTL component.**

- CDC synchronizers, reset synchronizers, and other low-level primitives are in
  `${IP_COMMON_PATH}/rtl/verilog/` and `${IP_COMMON_PATH}/rtl/vhdl/`. Instantiate them
  directly rather than re-implementing them.
- If a new primitive would be reusable across other IPs, write it in
  `${IP_COMMON_PATH}/rtl/` (both verilog/ and vhdl/) rather than inside this IP's `design/rtl/`.
- Instantiation paths must use `IP_COMMON_PATH`, never a hardcoded path.

## Coding Standards

**RULE — Read both style guides in full before writing a single line of RTL.**

- `.agents/VerilogCodingStyle.md` — lowRISC Comportable style for SystemVerilog
- `.agents/VHDL2008CodingStyle.md` — IEEE 1076-2008 VHDL style

**RULE — These documents are immutable. Do not edit, summarize, or override them.**
Every rule in these guides applies to every `.sv` and `.vhd` file produced by this step.
If a style-guide rule and any other instruction conflict, the style guide wins for all
matters of code formatting and structure.

**RULE — No silent deviations.** If a synthesis constraint or language limitation makes
strict compliance impossible at a specific line, add an inline comment at that exact line
explaining why. Deviations without a comment are a quality-gate failure.

**RULE — No SystemVerilog `interface` constructs.** Icarus Verilog does not support SV
`interface` / `modport` / `virtual interface`. Every inter-module connection must use
explicit individual `input` / `output` ports. This applies to the register-access bus,
any other internal buses, and all testbench connections. Using `interface` is a
quality-gate failure.

## Design Architecture

Each IP is structured as **four independent top-level entities**, one per bus protocol.
All four share identical core logic and an identical register file. The hierarchy is:

```
IP_NAME_<proto>          ← top-level entity (one per protocol; IP-local)
├── claude_<proto>_if    ← bus-to-regfile bridge (SHARED — lives in common/)
├── IP_NAME_regfile      ← register file (shared, identical in all four tops)
└── IP_NAME_core         ← core logic (shared, may be split across multiple files)
```

Supported protocols and their top-level entity names:

| Protocol    | Top-level entity   | Bus bridge (from common/)  |
|-------------|--------------------|-----------------------------|
| AHB-Lite    | `IP_NAME_ahb`      | `claude_ahb_if`             |
| APB4        | `IP_NAME_apb`      | `claude_apb_if`             |
| AXI4-Lite   | `IP_NAME_axi4l`    | `claude_axi4l_if`           |
| Wishbone B4 | `IP_NAME_wb`       | `claude_wb_if`              |

**RULE — Never write per-IP `IP_NAME_<proto>_if` files.** The bus-to-regfile bridges are
shared common components in `$IP_COMMON_PATH/design/rtl/verilog/` and
`$IP_COMMON_PATH/design/rtl/vhdl/`. Always instantiate `claude_<proto>_if` from common.

### Layer responsibilities

| Layer | Files | Allowed logic |
|-------|-------|---------------|
| **Top-level** (`IP_NAME_<proto>`) | `IP_NAME_<proto>.sv/.vhd` (IP-local) | Port declarations and submodule wiring **only**. Zero logic. |
| **Bus bridge** (`claude_<proto>_if`) | `$IP_COMMON_PATH/design/rtl/verilog/claude_<proto>_if.sv` | Protocol handshake, address decode, write-enable generation, read-data mux. No functional IP logic. |
| **Register file** (`IP_NAME_regfile`) | `IP_NAME_regfile.sv/.vhd` | Storage of all addressable registers. Generated by Step 2; do not modify except to fix parse errors. |
| **Core logic** (`IP_NAME_core` + helpers) | `IP_NAME_core.sv/.vhd` + optional additional files | All functional IP behaviour. No bus protocol awareness. |

### Internal register-access interface

The bus interface drives the register file through a protocol-agnostic register-access bus.
**These signals are individual ports on each module — NOT a SystemVerilog `interface`.**
Using `interface`, `modport`, or `virtual interface` is forbidden (Icarus Verilog does not
support them and they are prohibited by the global rule in `Claude_IP.md`).

Every `IP_NAME_<proto>_if` must present this exact set of individual signals to
`IP_NAME_regfile`:

| Signal | Direction (if → regfile) | Width | Description |
|--------|--------------------------|-------|-------------|
| `reg_addr` | → | `ADDR_W` bits | Word address of target register |
| `reg_wdata` | → | `DATA_W` bits | Write data |
| `reg_wstrb` | → | `DATA_W/8` bits | Byte write-enable strobes |
| `reg_we` | → | 1 | Write enable (active-high, single cycle) |
| `reg_re` | → | 1 | Read enable (active-high, single cycle) |
| `reg_rdata` | ← | `DATA_W` bits | Read data returned the following clock |
| `reg_error` | ← | 1 | Address decode error |

This interface is identical regardless of which bus protocol is in use. The register file
sees the same signals from AHB, APB, AXI4-Lite, or Wishbone.

## Responsibilities

### 1. Core logic

Implement the functional IP behaviour — this is the only place real work happens:

- `design/rtl/verilog/IP_NAME_core.sv` — SystemVerilog implementation.
- `design/rtl/vhdl/IP_NAME_core.vhd` — VHDL-2008 implementation.

If the design is complex, split the core into multiple files. Each additional file must
follow the naming convention `IP_NAME_<function>.sv/.vhd` (e.g., `IP_NAME_datapath.sv`,
`IP_NAME_fsm.sv`). Document each split file in the Outputs table.

The core interfaces only with `IP_NAME_regfile` through the register-side ports defined in
`IP_NAME_reg_pkg`. It has no knowledge of the bus protocol above it.

### 2. Bus-to-regfile bridges (use from common — do NOT create per-IP copies)

The bus bridges live in `$IP_COMMON_PATH/design/rtl/` and are shared across all IP blocks:

| SV file | VHDL file | Protocol |
|---------|-----------|----------|
| `common/design/rtl/verilog/claude_apb_if.sv` | `common/design/rtl/vhdl/claude_apb_if.vhd` | APB4 |
| `common/design/rtl/verilog/claude_ahb_if.sv` | `common/design/rtl/vhdl/claude_ahb_if.vhd` | AHB-Lite |
| `common/design/rtl/verilog/claude_axi4l_if.sv` | `common/design/rtl/vhdl/claude_axi4l_if.vhd` | AXI4-Lite |
| `common/design/rtl/verilog/claude_wb_if.sv` | `common/design/rtl/vhdl/claude_wb_if.vhd` | Wishbone B4 |

Each bridge:
- Accepts the full bus protocol signals as ports on one side.
- Drives the register-access interface (`wr_en / wr_addr / wr_data / wr_strb / rd_en / rd_addr / rd_data`) on the other.
- Contains **only** bus protocol logic — handshake state machines, address decode, strobe
  generation, read-data multiplexing, and protocol error signalling.
- Contains **no** functional IP logic whatsoever.
- Is parameterized by `DATA_W` (default 32) and `ADDR_W` (default 4).

**RULE — Do not create `IP_NAME_<proto>_if` files in `design/rtl/`.** Instantiate the
shared `claude_<proto>_if` from common instead.

### 3. Top-level entities

Implement one top-level per protocol:

- `design/rtl/verilog/IP_NAME_ahb.sv` / `design/rtl/vhdl/IP_NAME_ahb.vhd`
- `design/rtl/verilog/IP_NAME_apb.sv` / `design/rtl/vhdl/IP_NAME_apb.vhd`
- `design/rtl/verilog/IP_NAME_axi4l.sv` / `design/rtl/vhdl/IP_NAME_axi4l.vhd`
- `design/rtl/verilog/IP_NAME_wb.sv` / `design/rtl/vhdl/IP_NAME_wb.vhd`

Each top-level:
- Declares all I/O ports (bus protocol signals + any IP-specific I/O).
- Instantiates exactly three submodules: `claude_<proto>_if` (from common), `IP_NAME_regfile`,
  `IP_NAME_core`.
- Contains **no logic** — only port declarations, parameters/generics, and submodule
  instantiations with named port connections.

### 4. Behavioral reference model

Write `design/behavioral/IP_NAME_model.sv` — a non-synthesisable functional model of the
IP core used by the UVM scoreboard in Step 6. It models the expected response to any
sequence of register reads and writes, independent of bus protocol.

### 4b. Core logic pitfalls (timer-class IPs)

**`ctrl_restart` priority over `ctrl_en`**: In the counter `always_ff` / `process` block,
branch priority matters. If `ctrl_en = 0` and `ctrl_restart = 1` simultaneously, the counter
must stop — the enable-off condition takes priority. Incorrect implementation:

```systemverilog
// WRONG: ctrl_restart can bypass the ctrl_en=0 stop condition
end else if (ctrl_restart && active_q) begin
  count_q <= safe_load_val;

// CORRECT: guard ctrl_restart with ctrl_en
end else if (ctrl_en && ctrl_restart && active_q) begin
  count_q <= safe_load_val;
```

The prescaler typically has this right already (`if (!ctrl_en)` check comes before restart).
Ensure the counter matches the prescaler's priority ordering.

**`safe_load_val` — minimum load of 1**: When LOAD=0, a down-counter underflows every cycle
in repeat mode, causing a continuous interrupt storm. Guard with:

```systemverilog
logic [DATA_W-1:0] safe_load_val;
assign safe_load_val = (load_val == '0) ? {{(DATA_W-1){1'b0}}, 1'b1} : load_val;
```

Use `safe_load_val` everywhere the counter is reloaded. Document this behavior in the
LOAD register description: "LOAD=0 is treated as 1 in repeat mode."

This also affects testbench COUNT read-only tests — see `directed_tests.md` for the
read-before/write/read-after pattern that avoids hardcoded expected values.

### 5. Style and documentation requirements

- Every port in every module/entity is documented with an inline comment.
- Every `always_ff` / `process` block has a one-line comment describing its purpose.
- Parameterize data width, address width, and number of registers via SV `parameter` /
  VHDL `generic`. No hardcoded widths anywhere.
- Reset is synchronous active-low. Reset polarity is a top-level `parameter`/`generic`.
- SV and VHDL implementations are port-for-port identical for the same module name.

### 6. Parse checks

Run before marking complete:

```bash
# SystemVerilog (claude_*_if comes from common — not from IP design/rtl/)
iverilog -tnull -g2012 \
    design/rtl/verilog/IP_NAME_reg_pkg.sv \
    design/rtl/verilog/IP_NAME_regfile.sv \
    design/rtl/verilog/IP_NAME_core.sv \
    "${IP_COMMON_PATH}/design/rtl/verilog/claude_<proto>_if.sv" \
    design/rtl/verilog/IP_NAME_<proto>.sv   # repeat for each proto

# VHDL-2008
ghdl -s --std=08 design/rtl/vhdl/IP_NAME_reg_pkg.vhd
ghdl -s --std=08 design/rtl/vhdl/IP_NAME_regfile.vhd
ghdl -s --std=08 design/rtl/vhdl/IP_NAME_core.vhd
ghdl -s --std=08 design/rtl/vhdl/IP_NAME_<proto>_if.vhd
ghdl -s --std=08 design/rtl/vhdl/IP_NAME_<proto>.vhd  # repeat for each proto
```

All files must pass before moving to Step 4.

## Quality Gate

- `iverilog -tnull` passes on every `.sv` file.
- `ghdl -s --std=08` passes on every `.vhd` file.
- SV and VHDL top-levels are port-for-port identical for the same protocol.
- All four top-levels instantiate exactly `claude_<proto>_if` (from common), `IP_NAME_regfile`,
  and `IP_NAME_core`. Nothing more, nothing less.
- No per-IP `IP_NAME_<proto>_if` files exist in `design/rtl/` — the common bridges are used.
- No functional logic in any top-level or bus bridge file.
- No bus protocol signals reach `IP_NAME_core` — they are fully absorbed by `claude_<proto>_if`.
- No hardcoded widths; all widths are driven from parameters/generics.
- All style-guide rules followed (no tabs, correct naming, explicit widths, etc.).
